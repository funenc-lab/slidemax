# AGENTS.md - Reference Directory Guide

This directory stores supporting references for `skills/slidemax_workflow/`.

## Authority Boundary

- `../AGENTS.md` is the only canonical workflow handbook.
- `docs/` contains topic-specific reference notes and compatibility redirects only.
- This directory must not become a second workflow handbook, alternate role handbook, or a source of canonical stage/command rules.

## What Belongs Here

- Narrow reference notes for image setup, format guidance, SVG handling, and related topics
- Compatibility redirect pages that point readers back to `../AGENTS.md`
- Repository structure notes that explain boundaries without redefining workflow stages

## What Does Not Belong Here

- A duplicated end-to-end workflow handbook
- Alternate stage order definitions
- Canonical role-routing rules
- Canonical workflow-stage or command-order rules that disagree with `../AGENTS.md`

## Update Rules

1. If a change affects workflow stages, command order, or delivery rules, update `../AGENTS.md` first.
2. If a change affects role routing, update `../roles/AGENTS.md` and the relevant role files.
3. Keep `docs/README.md` and `docs/README_CN.md` as thin redirect entries.
4. Keep older handbook filenames as redirect pages only when compatibility matters.
